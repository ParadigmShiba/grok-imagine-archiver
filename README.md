# Grok Imagine Archiver 🖼️

This project is a fully automated tool designed to **intercept, collect, and elegantly display** the images and prompts generated by users on `grok.com` (Imagine model).

The core focus of the project is a dual-track approach: **Background WSS Interception** and a **Premium Web UI Gallery**.

---

## 🌟 Core Features
1. **Anti-Bot CDP Interceptor** (`scraper.py`)
   - Launches a clean, isolated Chrome container using Chrome DevTools Protocol (CDP). Through Dynamic Port allocation and Standalone Process mechanisms, it ensures stable mounting across various Windows environments, effectively outsmarting Cloudflare's bot defenses.
   - Operates in a manual interception mode where the script listens to your browsing session (`scraper.py`).
   - Dynamically recursively binds all tabs (including future ones), continuously listening to `wss://grok.com/ws/imagine/listen`.
   - **Smart Session Packaging**: Accurately uses `request_id` to automatically group every image and `prompt.txt` from a single Prompt generation into an isolated folder (`gallery/{request_id}_{timestamp}`). This prevents edge cases such as being blocked by moderation or generating zero images.
   - **Native Video Interception**: Supports intercepting "Download" button events from Grok's history gallery, automatically capturing `.mp4` streams and persisting them to `video-gallery/`.
2. **Modern Visual Gallery** (`api.py` & `frontend/`)
   - **Auto-Polling Feed**: Employs a vertical layout where a single conversation spans the entire row. The system checks the background status every 3 seconds and seamlessly re-renders the screen when new images are available.
   - **Zero-Latency Experience & Real-time Filtering**: The backend uses an incremental JSON caching system based on `mtime`, allowing tens of thousands of records to load within `0.02 seconds`. The interface provides pure frontend dynamic string search and an instant toggle to "Hide items with no images".
   - **Cinematic Viewer Mode (Pan & Zoom)**: Clicking an image instantly enters a 75vw full-bleed immersive experience with no text. It supports free zooming (0.2x~10x) using the mouse scroll wheel, and "Right-click and drag" to pan the canvas and view key details.
   - **One-Click Copy**: An elegant copy button is placed next to each Prompt, providing visually appealing feedback upon clicking, making it easy to re-feed the prompt back to Grok for fine-tuning.
   - **Performance Mandate**: Fully implements Event Delegation, abandoning pointless DOM event listeners, ensuring zero memory leak overhead.

## 🚀 Quick Start

### 1. Install Environment and Dependencies
Ensure you have Python 3.10+ installed. Open your terminal and execute the following commands to clone the repository and set up the environment:
```powershell
git clone https://github.com/ParadigmShiba/grok-imagine-archiver.git
cd grok-imagine-archiver
python -m venv venv
.\venv\Scripts\activate
pip install -r requirements.txt
playwright install
```
*(If `requirements.txt` does not exist yet, you will need to install `playwright`, `fastapi`, `uvicorn`)*


### 2. Start the Interceptor & Web UI Gallery (Recommended)
```powershell
.\venv\Scripts\python scraper.py --start-gallery
```
- This command opens an isolated browser based on `chrome_profile/` and instantly launches the Web UI gallery on `http://localhost:8000` in a new tab.
- Please log in to `grok.com` in this browser and start generating images.
- The terminal will provide real-time updates on how many images have been successfully captured and saved to `gallery/`.

### 3. Start the Web UI Gallery Standalone (Optional)
If you only wish to view the gallery without starting the scraper interceptor:
```powershell
.\venv\Scripts\python -m uvicorn api:app --reload --port 8000
```
- Open a browser and navigate to: [http://localhost:8000](http://localhost:8000)
*(If port 8000 is occupied or blocked by Windows, you can change `--port 8000` to an available port like `8080`)*
- You will see a gallery arranged in a vertical "Feed", where each row represents the complete output result of a single conversation.

## 🛠 Directory Structure

```text
grok-imagine-archiver/
├── api.py            # Web UI Backend (FastAPI)
├── scraper.py        # WSS Manual Interceptor
├── docs/             # Development and Architecture Documentation
├── frontend/         # Gallery Frontend Project (HTML/CSS/Vanilla JS)
├── gallery/          # Captured Image Storage (Grouped by request_id)
├── video-gallery/    # Captured MP4 Video Storage
├── gallery_cache.json# Incremental Cache ensuring blazing fast API load times
├── chrome_profile/   # Chrome Login Info Cache (Do not commit to version control)
└── venv/             # Python Virtual Environment
```
